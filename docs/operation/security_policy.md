# AI Workflow Security Policy

## 目的

AI業務フローにおいて、患者情報・医療情報・契約情報・未公開特許情報・APIキー等の機微情報を、誤って外部送信・GitHub保存・公開しないための基本方針を定める。

## 情報分類

### Public

外部公開可能な情報。

例：

- 公開済みWebサイトの内容
- 公開済みLPの文言
- 一般的な事業説明
- 公開済みSNS投稿案

### Internal

社内利用に限定する情報。

例：

- 業務フロー案
- GitHub Issue案
- 社内タスク整理
- 未公開のWeb改善案

### Confidential

外部送信・GitHub保存に注意が必要な情報。

例：

- 契約交渉
- 投資家対応の詳細
- 開発会社との詳細なやり取り
- 非公開の事業計画

### Highly Confidential

原則としてGitHubに保存しない情報。

例：

- 患者個人情報
- 医療鑑定の具体資料
- 弁護士相談内容
- 未公開特許の技術核心
- APIキー・パスワード
- NDA対象情報

## AIツール利用方針

### Claude Code / Codex CLI

非機密用途に限定する。

使用可：

- コード作成
- リポジトリ構造整理
- 公開済みWebサイト修正
- 非機密ドキュメント整備

使用不可：

- 患者情報
- 医療鑑定資料
- 契約交渉詳細
- 未公開特許情報
- NDA対象情報

### APIキー利用スクリプト

scripts/analyze_thread_api.py など、APIキーで制御するスクリプトは、機微情報を含む可能性がある処理にも使用可能とする。

ただし、以下を守る。

- GitHubへ自動保存しない
- 出力を人間がレビューする
- outputs ディレクトリはGit管理外とする
- 必要な要約のみを非機密化して docs や Issue に反映する

## GitHub保存禁止

以下はGitHubに保存しない。

- APIキー
- .env ファイル
- 患者個人情報
- 医療鑑定の具体資料
- 契約書原本
- NDA対象資料
- 未公開特許の技術詳細

## inbox利用ルール

inbox ディレクトリには、原則として非機密化したスレッド本文または要約のみを保存する。

以下は保存しない。

- 患者名
- 生年月日
- 医療機関名と症例情報が結びつく具体情報
- 弁護士相談の具体内容
- 契約交渉の詳細
- 特許出願前の技術核心
- APIキー・パスワード等

## コミット前チェック

コミット前に以下を確認する。

- git status
- git diff
- git diff --cached

以下が含まれていないことを確認する。

- APIキー
- 患者名
- 医療機関の個別症例情報
- 契約書本文
- 未公開特許の核心技術
- パスワード・トークン

## 原則

迷った場合はGitHubに入れない。

公開してよいか不明な情報は、まずローカルの outputs ディレクトリに留める。

AI生成物はすべて案であり、人間レビューを必須とする。
